## Why 1024

### The Meaning (Deep)

**1024 is the foundation of all computing.**

- `1024 = 2^10`: ten doublings from 1 to 1024
- `1024 bytes = 1 KB`: the building block of all storage
- `1024 × 1024 = 1 MB`: it compounds
- Every memory page, every texture, every buffer starts at 1024
- It's the number that makes everything else possible

**For developers, 1024 is instinctive.** They don't calculate it: they *feel* it. It's the number that means "foundation, power of two, compounding growth."

### The Metaphor

> **8090.ai** named themselves after a generation (80s-90s kids).  
> **1024** is named after a *universal constant*: the number that powers every computer ever built.

Just like 1024 is the foundation of all data, **1024 Studio** is the foundation of all software projects.

## VIRAL LOOPS

### Loop 1: Team Invite Chain (Most Powerful)

```
User A (free) invites User B
    → Both get 1 week Team trial
        → User B invites User C
            → All three get 2 weeks Team trial
                → Team converts → A gets 3 months free
```

**Implementation:**
- When a free user creates a project, prompt: "Invite your team to collaborate"
- Each team member who joins extends everyone's trial by 3 days
- Trial converts to paid after 14 days (or extends with more invites)

### Loop 2: Template Share Loop

```
User creates a template → Shares on marketplace
    → Template includes "Built with 1024" badge
        → Template buyers see the badge
            → Badge links to 1024 download
                → New user installs → Template creator gets credits
```

### Loop 3: Export Share Loop

```
User exports project to PDF/Markdown
    → PDF includes "Generated by 1024" footer
        → Recipient sees the footer
            → Footer links to 1024 download
                → New user installs → Exporter gets credits
```

### Loop 4: AI Demo Loop

```
User shares AI-generated requirements on Twitter
    → "Just generated 15 requirements in 30 seconds with @1024studio 🤯"
        → Followers click link
            → Landing page shows live demo
                → New user signs up → Sharer gets 100 credits
```

## REFERRAL DATA MODEL

```sql
CREATE TABLE referral_codes (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    code TEXT UNIQUE NOT NULL,        -- "LAKSH-1024"
    url TEXT NOT NULL,                 -- "1024studio.ai/r/LAKSH-1024"
    uses INTEGER DEFAULT 0,
    max_uses INTEGER DEFAULT -1,      -- -1 = unlimited
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE referral_events (
    id TEXT PRIMARY KEY,
    referrer_user_id TEXT NOT NULL,
    referee_user_id TEXT NOT NULL,
    referral_code TEXT NOT NULL,
    event_type TEXT NOT NULL,          -- 'signup', 'pro_upgrade', 'team_upgrade', 'enterprise_upgrade'
    credits_earned INTEGER DEFAULT 0,
    reward_granted TEXT DEFAULT '',
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE user_credits (
    user_id TEXT PRIMARY KEY,
    balance INTEGER DEFAULT 0,
    lifetime_earned INTEGER DEFAULT 0,
    lifetime_spent INTEGER DEFAULT 0,
    referral_tier TEXT DEFAULT 'seed', -- seed/sapling/tree/forest/ecosystem
    referral_count INTEGER DEFAULT 0,
    updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE credit_transactions (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    amount INTEGER NOT NULL,           -- positive = earned, negative = spent
    reason TEXT NOT NULL,              -- 'referral', 'purchase', 'daily_streak', 'template_submit'
    reference_id TEXT DEFAULT '',      -- referral_event_id or purchase_id
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE referral_leaderboard (
    user_id TEXT PRIMARY KEY,
    display_name TEXT NOT NULL,
    referral_count INTEGER DEFAULT 0,
    tier TEXT DEFAULT 'seed',
    badge_emoji TEXT DEFAULT '🌱',
    joined_at TEXT DEFAULT (datetime('now'))
);
```
